**Baseline** (`vpxor` + `vpblendw`):

```asm

	.cfi_startproc
# %bb.0:
	vpxor	%xmm1, %xmm1, %xmm1
	vpblendw	$238, %ymm1, %ymm0, %ymm0       # ymm0 = ymm0[0],ymm1[1,2,3],ymm0[4],ymm1[5,6,7],ymm0[8],ymm1[9,10,11],ymm0[12],ymm1[13,14,15]
```

File: [`baseline/combine_and_pshufb.s`](baseline/combine_and_pshufb.s)

**Patched** (`vpshufb` + constant-pool mask):

```asm
combine_and_pshufb:                     # @combine_and_pshufb
	.cfi_startproc
# %bb.0:
	vpshufb	.LCPI0_0(%rip), %ymm0, %ymm0    # ymm0 = ymm0[0,1],zero,zero,zero,zero,zero,zero,ymm0[8,9],zero,zero,zero,zero,zero,zero,ymm0[16,17],zero,zero,zero,zero,zero,zero,ymm0[24,25],zero,zero,zero,zero,zero,zero
	retq

```

## Interpretation

- **Instruction count:** patched emits 1 fewer static instruction (3 ‚Üí 2, including `ret`).
- **Shuffle uops in source:** 2 ‚Üí 1 (`vpxor`+`vpblendw` replaced by one `vpshufb`).
- **Function code bytes:** patched function body is 1 bytes smaller in `.text`.
- **Constant-pool tradeoff:** patched introduces `.LCPI0_0` ‚Äî a 32-byte AVX2 PSHUFB control mask in section `.rodata.cst32` (`.p2align 5`). Baseline uses immediate-only `vpblendw` ($0xEE) with no pool.
- **Linked content sections (`llvm-size -A` total):** patched is 31 bytes larger (59 ‚Üí 90 B), driven mainly by the 32-byte mask plus a 1-byte `.text` saving.
- **Total object file:** patched `.o` is 256 bytes larger on disk (800 ‚Üí 1056 B), including extra ELF metadata (`.rela.text` for the RIP-relative `vpshufb`).

---

## Limitations

- Measurements are **static** (object file / disassembly), not runtime dynamic instruction count under inlining or LTO.
- Isolated single-function object: no linker layout, no hot/cold splitting, no I-cache line packing effects.
- `llvm-mca` microarchitectural cost **not** evaluated in Step 4A1.
- Baseline generated by temporarily stashing the backend patch and rebuilding the same `llc` binary; patched restored immediately afterward.
- Object inspection uses Homebrew LLVM 22 tools on objects emitted by LLVM 17.0.6 `llc`.
- Constant-pool size counts `.byte` payload; section alignment padding in `.rodata` may add bytes beyond the 32-byte mask.
